Finite State Machines

A FSM is a mathematical model of computation that describes a system's behavior as a set of states and transitions.
A FSM describes the behaviour of a Regular Expression (RegEx). Each FSM is composed of the following 5-tuple (Q, Σ, δ, q0, F):

  • a finite set of states
  • Σ: a finite alphabet set
  • δ: transition function - a set of maps from states and inputs into states
  • q0: an initial state
  • F: a set of final/accepting states
    with line breaks.

There are 2 special states: Start and Finish

Start State : The FSM always begins in this state. It is usually represented with an arrow pointing at it.

Finish State : The FSM must finish in this state. There can be multiple. Finish states are those where the machine accepts the input.Accepting states are usually represented by double circles.

A transition is a connection between two states that describes the machine's behavior.

All accepted strings for FSM’s should be ordered lexicographically.

Determinstic Finite State Automata (DFA)
A deterministic finite automata (DFA) is a type of FSM that has only one transition for each label in a state.
They are machines in which each transition is uniquely determined by the current state and input

Non-deterministic finite automata (NFA)
A NFA allows for multiple transitions with the same label for many states.
A NFA has more than one entry in at least one cell of the State Transition Table.

  • Both NFA and DFA only recognise regular languages
  • DFA and NFA have distinct definitions, but it can be shown that they are equivalent. That is, for any NFA we can construct an equivalent DFA and vice versa
