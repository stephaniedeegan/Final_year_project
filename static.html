Static Methods

A java method is a bundle of lines of code with a common function. Methods also define the behaviour of Classes in object oriented programming.

Writing methods allows us to maintain important code or algorithms that are used in repeatedly in different places in our applications.

A method must be written or declared before it can be used. After it is declared it is used by calling or invoking it in your program.

Method Declaration:
Access_modifiers return_type method_name( formal_parameter_list)
  {
     statements;
   }

Example
public static void printHello ( )
  {
     System.out.println(“Hello world”);
  }

Formal Method Declaration:
access_modifiers return_type method_name (formal parameter list)
  {
    statements; // if the return type is not void then you need a return statement.
   }

Example
public static int addTwoNumbers ( int x, int y )
  {
    return x + y;
  }
The int here declares that we want to return an int.

Rules for Declaring Methods

  • The 1st line of tells the programmer how the method should be used.
    → Method Signature
  • The number of parameters in the call to the method must match the type and number of parameters in the method declaration
  • If the return type is not void, then the method statements must contain the return statement
  • Method name have to obey rules for legal identifiers. As a code convention rule, the 1st letter in the identifier is a small letter, and the remaining words hvae their 1st letter captialised.
Scope  : is the portion of the program from which the variable can be addressed.
There are 2 types of scope  → block and class

A variable that is defined within a method is called a local variable.
The variable declared in a for statement only extends to the structure of that statement.
You can't declare two variables with the same name in the same method as their scopes overlap.

Static Variables  : Methods and attributes which are Static have one “version” of them.
public and private are called Access Modifiers.
 → private methods can NOT be seen by other classes

You can define two or more methods within the same class that share the same name, as long as their parameter declarations are different.  →These are called overloaded methods
