Classes and Objects

Class Scope : This is when a variable (attribute) is declared just inside the first line of a class after the first brace {. The scope of this variable covers the whole class which includes any methods inside the class.

Data Encapsulation : is the process of hiding implementation details, which is achieved by bundling the methods and variables of a class reprsentaion in the class.

Private : This is an access modifier which is put in front of your class variables declarations. Only methods defined inside a class definition can have access to or are able to change the value of the variable.

Public : This allows other code to call these methods using an instance of an object and the dot “.” operator followed by the method name. You allow the public to have a way of accessing your attributes.

Information Hiding : only allowing the methods defined within a class to have access to or change the values of the attributes of the class. Achieved by declaring the class attribute variables to using the access modifier private in front of their declarations.

Inheritance : is the process of creating a new class from an existing class. It is a very important ability because it allows the programmer to develop and extend software which is dependent on previously proven super classes.

Overriding Methods : When a method is overridden in a subclass, and that method is invoked on an instance of the subclass, it is the overridden version of the method that is executed. The subclass method must have the same name and parameter types as the superclass method it overrides.

Method Overloading : Creating multiple versions of the same method with different parameters.

Writing Class constructor methods

  • The Class constructor method name is the same as the name of the Class itself
  • The constructor method never returns a value or type although it appears to do so
  • The constructor needs to be accessible from outside the class
       → Needs to be public
  • You can have multiple versions of the class constructor method
       → Method overloading
The key word this followed by the dot “.” operator allows you to specify which variable you are referring to explicitly.
        this.x refers to the class instance variable x

Example Code :
public class Point {
  private int x;
  private int y;
  public Point(int x, int y) {
    this.x = x;
    this.y = y;
  }
  public int getX() {
    return x;
  }
  public void setX(int x) {
    this.x = x;
  }
  public int getY() {
    return y;
  }
  public void setY(int y) {
    this.y = y;
  }
  public void printPoint() {
    System.out.println("Point coordinates are (" + x + "," + y + ")");
 }
}
