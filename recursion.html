Recursion

Recursion is the process in which a function calls itself directly or indirectly. It can reduce the length of one's code, making it easier to read and write.

All recursive methods must have a base case. A recursive method keeps calling itself until it reaches the base case and then it filters back up.

Direct recursion : functions call themselves
Indirect recursion : functions call other functions to call the original function

A recursive method should not call itself more than once.
When you run a program, the computer creates a stack for you.
Each time a method is called, you push the method on the stack.
Each time the method returns or exits, you pop the method off the stack.
If a method calls itself recursively, you just push another copy of the method onto the stack. This is a way to visualise how recursion works.

Simple Factorial Example
Factorial(n) = n * Factorial (n-1) (for n>1) // recursive step
Factorial(1) = 1// base case or anchor
Factorial(0) = 1

Sigma Example
public static int mySigma(int x) {
 if (x <= 0) {  // base case
   return 0;}
 else {  // recursive step
   return (x + mySigma(x-1));}
}

Advantages of Recursion

  • Leads to simpler, shorter, and more elegant code, making it easier to understand
  • It is useful for solving complex problems

Triangular Numbers Example
Iterative Way
public static int triangleI (int n)
 {   int sum = 0;
   for(int i=1; i<=n; i++)
    {
       sum = i + sum;
    }
     return sum;
  }
Recursive Way
public static int triangleR (int n)
 {  
  if (n == 1)  //Base Case
  {
   return 1;
  }
  else
  {
     return (triangleR (n-1) + n);
  }
 }

Recursion vs Iteration
Recursion

  • Shorter code
  • Terminates when base case becomes true
  • Used with functions
Iteration
  • Longer code
  • Terminates when the condition becomes false
  • Used with loops
