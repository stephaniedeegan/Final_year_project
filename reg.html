Regular Expressions

Regular Expressions are routinely used for validating input and retrieving information. They are a concise means of matching strings.
The String class using matches() method :

  • The general format for the matches method is “Problem-string”.matches(“RegEx”)
  • “abc”.matches(“abc”) - returns true
  • “Abc”.matches(“abc”) - returns false
  • “aabbaba”.matches(“a[ab]*”) - returns true

RegEx Grammar

  • [ ] - represents alternatives/or
    (e.g. ba[trn] - bat,bar,ban accepted, but bad not valid)

  • ^ - represents NOT / used to match the beginning of a line
    (e.g. ca[^trn] - cat,car,can not accepted, but cab is valid)

  • Ranges [a-z]
    (e.g. [A-Z] - any captial letter from A-Z)
    • (e.g. [0-9]&&[^2]] 0-9 but Not 2)

  • * - zero or more times
    (e.g. xy* - x,xy,xyy,xyy...)
    • (e.g. [as]* - aa,,aassa,assa...)

  • + - one or more times
    (e.g. ab* - ab,abbb,abbbb...)
    • (e.g. [0-9]+ - any sequence of >= 1 digits)

  • ? - zero or once (optional)
    (e.g. Colou?r - Colour Color)

  • Counted items { }
    (e.g. s{3} - sss only)
    • (e.g. s{2,} - 2 or more ss,sss...)
    (e.g. [a-z]{2,3} - any 2/3 letters, be,bed)

  • . - the wild character allowing for any character in a string, except the new line character
    (e.g. b.d - bed,bad,bud...)

  • / - We use backslash so that the following character is included. It is a special character as it means, do NOT treat the following character in the normal way
    (e.g. // - matches the quote / mark)
    • (e.g. /" - matches the quote “ mark)
    (e.g. /s - white space)
    (e.g. /b - matches the empty string at the beginning or end of a word

    Examples
    1. Write a RegEx that matches any string that contains only letters (uppercase or lowercase)
      ^[a-zA-Z]+

    2. Write a RegEx that matches any string that contains a valid email address (only letters/digits accepted)
      ^[a-zA-Z0-9]+@[a-zA-Z0-9]+\.[a-zA-Z]{2,4}

    3. Write a RegEx that matches the words car,cap,bar,bap
      ^[bc]a[rp]
